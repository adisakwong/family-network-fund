# Family Network Fund - Frontend + Google Apps Script

โปรเจกต์นี้เป็นแอปพลิเคชันเว็บแอป (Web App) สำหรับ **"กองทุนเครือญาติเพื่อสุขภาพและพิธีศพ (Family Network Fund for Healthcare and Funeral)"** ที่ทำงานบนสถาปัตยกรรมแบบ Serverless ซึ่งหมายความว่าในส่วนของระบบหน้าบ้าน (Frontend) ถูกออกแบบมาให้เป็นโหมด Static Website ล้วนๆ ทำให้สามารถนำไปฝากโฮสต์ฟรี เช่น **GitHub Pages** หรือบริการใกล้เคียงได้ทันที 

ในขณะที่ระบบหลังบ้าน (Backend) ทั้งตารางข้อมูล การคัดกรอง ข้อมูลสรุป และที่เก็บไฟล์รูปภาพ จะรณการทำงานผ่าน **Google Apps Script** ภายใน Ecosystem ของ **Google Workspace** (Google Sheets, Google Drive) ได้ฟรีโดยสมบูรณ์

## ✨ ฟีเจอร์หลักและการอัปเดตล่าสุด (Key Features)

- **ระบบเข้าสู่ระบบด้วยรหัสประชาชน 13 หลัก:** ล็อกอินเข้าสู่ระบบอย่างปลอดภัยและรวดเร็ว
- **หน้าลงทะเบียนพร้อมข้อมูลเฉพาะบุคคลลึกขึ้น:** บันทึกข้อมูลที่ครบถ้วนทั้ง ชื่อ, นามสกุล, เบอร์โทร, ที่อยู่ปัจจุบัน, และฟิลด์ใหม่ **วัน/เดือน/ปีเกิด**, **โรคประจำตัว**, **ผู้รับผลประโยชน์** และ **ความเกี่ยวข้องทางสายเลือด**
- **วันที่ลงทะเบียนอัตโนมัติ (Auto-Register Date):** ระบบจะเก็บวันที่ปัจจุบันที่ทำรายการลงทะเบียนส่งไปบันทึกลงในหลังบ้านให้อัตโนมัติ (รวมถึงช่วยโชว์ในหน้าระดับโปรไฟล์)
- **ระบบถ่ายภาพบนเบราว์เซอร์ (Camera API Integration):** ผู้ใช้งานสามารถถ่ายรูปเซลฟี่จากกล้องสมาร์ทโฟนหน้า-หลังได้ทันทีผ่านระบบ WebRTC หน้าเว็บ ระบบจะนำภาพมาพรีวิว ประมวลผล บีบอัดเป็น Base64 แล้วส่งไปให้ระบบ Apps Script แปลงเป็นรูปภาพจัดส่งลงในโฟลเดอร์ Google Drive อย่างอัตโนมัติ
- **รักษารูปแบบเบอร์โทรและรหัสประชาชน (Leading Zeros Retention):** หมดปัญหารหัสบัตรที่ขึ้นต้นด้วยเลขศูนย์ (0) หายไปเมื่อบันทึกลงใน Google Sheets สคริปต์ใหม่เสริมความมั่นใจในการจัดเก็บให้รูปแบบ Text เต็มที่
- **ดีไซน์สวยงาม ทันสมัย แบบ Responsive:** หน้าตาเว็บแอปได้รับการปรับปรุงการใช้งานบนโทนสีแบบพรีเมียม Dark Blue Gradient ใช้งานง่าย ออกแบบมารองรับการแตะกดของขนาดหน้าจอมือถือ (Mobile-First) 100%
- **ไม่มีค่าใช้จ่ายรายเดือน:** ทุกเครื่องมือและพื้นที่ทำงานฟรี 100% ไม่มีลิมิตการเข้าใช้เบื้องต้นอย่างแน่นอน

## 🛠 โครงสร้างตารางใน Google Sheets (สำหรับผู้ดูแลระบบ)
เมื่อมีการลงทะเบียนครั้งแรก และหลังบ้านทำงานสมบูรณ์ สคริปต์จะสร้าง Sheet ที่ชื่อ `members` พร้อมวางตารางหัวข้อเรียงตามลำดับคอลัมน์ดังนี้ให้โดยอัตโนมัติ (คอลัมน์ A ถึง K):
1. `Register_date` (วันที่ลงทะเบียน)
2. `Person_id` (รหัสบัตรประชาชน 13 หลัก)
3. `Name` (ชื่อ)
4. `Surname` (นามสกุล)
5. `Address` (ที่อยู่)
6. `Phone` (เบอร์โทรศัพท์)
7. `Image_URL` (ลิงก์รูปภาพใน Google Drive - รองรับการทำ Thumbnail)
8. `Birthdate` (วัน/เดือน/ปีเกิด)
9. `Illness` (โรคประจำตัว)
10. `Beneficiary` (ผู้รับผลประโยชน์)
11. `Related` (ความเกี่ยวข้อง)

## 🚀 วิธีการติดตั้งและใช้งาน (Deployment Guide)

### ส่วนที่ 1: การตั้งค่า Backend (Google Apps Script)
1. ไปที่ [Google Sheets](https://docs.google.com/spreadsheets/) ของคุณ สร้าง Spreadsheet อันใหม่และตั้งชื่อตามต้องการ (เช่น "Family Network DB")
2. คลิกเมนูด้านบน `ส่วนขยาย` (Extensions) > เลือกเมนูย่อย `Apps Script`
3. คัดลอกโค้ดทั้งหมดจากไฟล์ `Code.gs` ที่อยู่ในโปรเจ็กต์นี้ ไปวางทับลงในสคริปต์ของหน้าต่างใหม่ แล้วบันทึกโค้ดให้เรียบร้อย
4. กดยืนยันการเผยแพร่โปรเจ็กต์:
   - มุมขวาบนหน้าจอ กดปุ่มสีฟ้า **"การทำให้ใช้งานได้"** (Deploy) > เลือก **"การทำให้ใช้งานได้รายการใหม่"** (New deployment)
   - บริเวณช่องซ้ายมือ "เลือกประเภท" ให้คลิกรูปเกียร์/ตั้งค่า แล้วเลือกติ๊ก **"เว็บแอป"** (Web app)
   - ใส่การตั้งค่าดังนี้:
     - **คำอธิบาย:** "Family Network API V1" (พิมพ์ชื่อใดก็ได้)
     - **เว็บแอป ทำงานในฐานะ:** เลือกให้เป็น **"ฉัน"** (Me)
     - **ผู้มีสิทธิ์เข้าถึง:** เลือกระดับที่เปิดกว้างที่สุดคือ **"ทุกคน"** (Anyone)
   - กดปุ่ม **การทำให้ใช้งานได้** (Deploy) ด้านท้าย
5. **(สำคัญมาก)** หากเป็นการสั่งรันและ Publish สคริปต์เป็นครั้งแรก ระบบของหน้าจอจะขอเข้าถึงสิทธิ์ (Permissions) จากเจ้าของอีเมล:
   - คลิกที่ปุ่ม **"ตรวจสอบสิทธิ์"** (Authorize access) เข้าไปเลือกชื่อบัญชี Google หรือ Gmail
   - คลิกเลือกเมนูชั้นสุด **ขั้นสูง** (Advanced) เลื่อนลงมาเพื่อกด **"ไปยังโครงการสคริปต์ (... UNSAFE)"** 
   - กดปุ่ม **ยอมรับ (Allow)** เพื่อเปิดไฟเขียวให้สคริปต์นี้มีอำนาจในการแทรกแถวใหม่ของชีตนี้ หรืออัปโหลดรูปลง Google Drive ต้นทาง
6. หลังการ Verify เสร็จ ให้คัดลอก **URL ของเว็บแอป** ที่ลงท้ายด้วยอักษร `/exec` เก็บให้เรียบร้อยเพื่อไว้ใช้ฝั่งหน้าตาเว็บไซต์

### ส่วนที่ 2: การตั้งค่า Frontend และนำเปิดระบบให้บริการบน GitHub Pages
1. เปิดไฟล์ `Index.html` จากแหล่งโปรเจ็กต์ด้วยตัว Text Editor ที่ถนัด
2. เลื่อนลงมาล่างสุด (ก่อนขอบเขต `</body>`) คุณจะพบกับบล็อก `<script>`:
   ```javascript
   // ใส่ URL Web App ของ Google Apps Script ของคุณที่นี่
   const GAS_WEB_APP_URL = 'YOUR_GOOGLE_APPS_SCRIPT_WEB_APP_URL';
   ```
3. ให้นำลิงก์ Web App (URL API ฝั่ง Backend ที่เพิ่งคัดลองเอาไว้) เข้ามาวางในเครื่องหมายโควต (Quotes) แทนที่ข้อความจำลองเลย 
4. อ้างอิงแหล่งโปรเจกต์ สร้างพื้นที่ Repository ใหม่บนบัญชี GitHub แสนสะดวกของเรา
5. พุช (Push) หรือเลือกนำไฟล์กลุ่มหน้าบ้านคือ `Index.html`, `style.css`, `script.js` และ `README.md` โยนเข้าไปวางใน Repository ดังกล่าวให้เรียบร้อย (ไม่จำเป็นต้องนำไฟล์ `Code.gs` ใส่ไปเพราะมันเป็นส่วนทำงานหลังบ้านแล้ว)
6. ไปที่แท็บ **Settings** ในหน้าของ Repository 
7. เลื่อนหาเมนู **Pages** ตามแถบด้านซ้ายสุด 
8. ภายใต้พื้นที่ของ **Build and deployment** ส่วนขั้ว Source ตั้งค่าคงเดิมคือ **Deploy from a branch** 
9. เลือกสาขา (Branch) ตัวหลักคือ `main` (หรือบางเคสเป็น `master`) และเลือกโฟลเดอร์ที่ `/ (root)` พอกดคลิก **Save** เสร็จแล้วก็รอกลไกซักครู่
10. GitHub Pages จะดำเนินการสร้างและจัดเตรียมลิงก์เข้าเว็บไซต์ เมื่อแถบสีเขียวพร้อมด้วยตัว URL ประจำโดเมนส่งเสียงให้เห็น ก็สามารถกดเข้าไปสำรวจหน้าเว็บไซต์ และคัดลอกลิงก์ให้สมาชิกตระกูลใช้งานกันได้ 100% เลยครับ!
